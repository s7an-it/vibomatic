# P1: Developer Testing SVC

## Role

A developer evaluating the svc pipeline by running it against a trivial product.

## Goals

- Understand every pipeline phase by seeing concrete artifacts.
- Validate that the progressive-narrowing model produces consistent output.
- Use the todo-api as a template for real projects.

## Needs

- Clear, minimal examples of each artifact type.
- A validation script that confirms structural correctness.

## Pain Points

- Abstract methodology docs without concrete examples.
- Pipelines that only make sense on large products.

## Technical Context

Comfortable with git, shell scripts, markdown. Reads specs fluently.
